무슨 의도로 만들었나

캐릭터를 무엇으로 만들든(실사 촬영·VRM·메타휴먼·AI 영상생성) 최종 산출은 비디오 클립 + 메타(manifest)로 통일된다. 핵심은 비디오 파일이 아니라 클립의 위치·조합·순서 = 상태머신이다. 박스 데모는 "자리·흐름 명세"이고, 같은 규격의 고품질 클립으로 교체하면 그대로 동작한다(도구 중립 교환 포맷, 오픈소스 지향).

파일 구조 (.nva)

manifest.json (두뇌) + clips/ (비디오들)
  • state — talking(말하기 안정 포즈) / animation(동작). 클립 + 포즈 + face_bbox(헤드토킹 위치)
  • transition — 포즈 간 이동 클립. entry_pose → exit_pose
  • scenario — 연출(상태/이벤트 + 대사 + 타이밍 시퀀스)
  • 레이어 — 배경 + 캐릭터(알파/크로마) + 헤드토킹

사용법

  1. 데모 로드 또는 .nva 열기로 시작
  2. +말하기 / +동작 → state 생성 → 우측 폼에서 클립 업로드 + 포즈·face_bbox·loop 설정
  3. +전환 → 포즈 간 클립(앉기/서기 등)
  4. +시나리오 → 단계(상태·대사·시간) 추가 = 연출 흐름
  5. 중앙 미리보기 버튼(상태/시나리오)으로 돌려보기
  6. ⬇ .nva 내보내기 (manifest + 클립을 단일 zip으로)

포즈 규칙 (seamless)

각 클립은 해당 포즈에서 시작·끝나야 이어 붙일 때 튐이 없다. 연결 가능 = A.exit_pose == B.entry_pose (다르면 둘을 잇는 transition 필요). 시나리오는 이 포즈 경로를 자동 탐색해 전환을 끼워 넣는다.

데모(시연)와의 관계

이 에디터로 만든 .nva를 데모(demo.html / kiosk-v3)가 cascade에 넣어 실시간 렌더한다. 에디터=제작, 데모=시연으로 분리.
