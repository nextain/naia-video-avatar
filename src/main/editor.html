naia video clip avatar — 비디오 클립 아바타 포맷 제작/편집 도구

무슨 의도로 만들었나

캐릭터를 무엇으로 만들든(실사 촬영·VRM·메타휴먼·AI 영상생성) 최종 산출은 비디오 클립 + 메타(manifest)로 통일된다. 핵심은 비디오 파일이 아니라 클립의 위치·조합·순서 = animations 풀 + scenario 그래프이다.

파일 구조 (.nva v0.2)

manifest.json (두뇌) + clips/ (비디오들)
  • animation — 재료 풀의 원소. 클립 + 포즈 + loop + can_talk + face_bbox. 종류는 조합에서 유도: 말하기=loop&can_talk / 대기·듣기=loop / 제스처=둘 다 off / 전환=entry_pose≠exit_pose
  • scenario — 노드 그래프(nodes/edges). start 노드가 가리키는 첫 scene = idle 진입점
  • 레이어 — 배경 + 캐릭터(알파 VP9 webm) + 헤드토킹(can_talk 의 face_bbox 영역만 Ditto)

사용법

  1. 데모 로드 또는 .nva 열기로 시작 (구 v0.1 파일은 자동 v0.2 변환)
  2. +말하기(loop+can_talk) / +동작(제스처) / +전환(포즈 이동) → 애니 생성 → 우측 폼에서 클립 업로드 + 포즈·face_bbox 설정
  3. scenario: +scene 으로 노드 추가(애니 지정) → edges 로 흐름 연결(start→…)
  4. 중앙 미리보기 버튼(애니/시나리오)으로 돌려보기 — 알파 클립은 배경 위에 합성됨
  5. ⬇ .nva 내보내기 (manifest + 클립을 단일 zip으로)

배경 · 알파

캐릭터 클립이 알파(VP9 yuva420p webm)면 배경 위에 그대로 합성된다(크로마키 불요). 배경 transparent = cascade 가 알파 스트림 → 소비자(TalkingKiosk web/naia-os)가 배경색 합성. 구형 불투명 클립은 chroma_key 로 투명 처리(하위호환).

cascade 연동 (실시간 발화)

이 .nva(v0.2)를 cascade(output_cascade/nva_loader.py)가 로드 → idle/event 클립 재생 + speaking 시 can_talk 애니의 face_bbox 영역을 Ditto 헤드토킹으로 오버레이 → 비디오 스트림.
